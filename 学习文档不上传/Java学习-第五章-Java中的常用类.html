StringBuffer类

String的问题

在Java中，String类是final类型的。是不可改变的，如果对一个String进行修改，只能创建新的字符串。之前的字符串则会进入垃圾回收，新字符串被指向新的地址。

解决

使用StringBuffer类（字符串缓冲区）操作字符串。

说明 StringBuffer类与String类的最大区别是他的内容和长度可变。StringBuffer类似一个字符容器，当在其中添加或删除字符时，所操作的都是这个字符容器，因此不会产生新的StringBuffer对象。

包装类

由于Java的8种基本数据类型不支持面向对象的编程机制（没有属性和方法）。然而在Java中，很多类的方法都需要接收引用数据类型的对象，此时无法将一个基本数据类型传入。

除Character外，包装类都有parseXxx(String s)&valueOf(String s)的静态方法,但注意需基本类型相同，否则编译通过，运行报错。

JDK8-新增日期与时间类

为满足更多需求，JDK8中新增了java.time包，在该包下包含更多日期和时间操作类。

类

类名 功能描述
Clock 用于获取指定时区的当前日期、时间
DayOfWeek 枚举类，定义了一周七天周一到周日的枚举类
Duration 表示具体持续时间，提供ofXxx()方法用于获取指定时间的时分秒等
Instant 表示具体时刻，可以精确到纳秒。提供了静态的now()方法来获取当前时刻。提供静态的now(Clock clock)方法来获取clock对应的时刻。同时提供一系列的plusXxx()方法阿里获取当前时刻基础上加上一段时间以及一系列的minusXxx()方法在当前时刻基础上减去一段时间。
LocalDate 表示不带时区的日期，2020-05-24。
LocalTime 表示不带时区的时间，11:37:21。
LocalDateTime 表示不带时区的日期时间。
Month 枚举类，定义1-12月的枚举值。
MonthDay 表示月日，如05-24。
Year 表示年。now()，plusYears()，minusYears()。
YearMonth 表示年月。
ZoneID 表示时区。
ZoneDateTime 表示时区化的日期、时间。

DateFormat类

用于格式化日期类。抽象类，不能被直接实例化。

常用方法

方法声明 功能描述
static DateFormat getDateInstance() 用于创建默认语言环境和格式化风格的日期格式器
static DateFormat getDateInstance(int style) 用于创建默认语言环境和指定格式化风格的日期格式器
static DateFormat getTimeInstance() 用于创建默认语言环境和格式化风格的日期时间格式器
static DateFormat getDateTimeInstance(int dateStyle,int timeStyle) 用于创建默认语言环境和指定格式化风格的日期时间格式器
String format(Date date) 将一个Date格式化为日期时间字符串
Date parse(String source) 将给定字符串解析成一个日期

常用格式

  • Full 用于表示完整格式
  • Long 用于表示长格式
  • Medium 用于表示普通格式
  • Short 用于表示短格式
