类的继承

特点

​ 只支持单继承，一个类只能有一个父类。

​ 可实现多层继承–继承多层属性？

​ 只能访问公共成员

优点

​ 简化代码，提高复用性，提高维护性。

缺点

​ 类和类间（子父类）的耦合性（关系）太强。

关键字

​ extends

特别的

空继承等于继承Object类（所有类的父类-根类）

example:
public Cat extends Animal{} //Cat继承Animal非私有方法，属性

重写父类方法

为什么要进行方法重写？

父类方法无法满足子类的具体需求

要求声明与父类方法声明完全一致。

注解example：

super关键字

用于访问父类成员(成员变量、成员方法、构造方法)

example:

super(); //访问父类同名方法

object类

object类是所有类的根类、基类。

当类没有被extends关键字修饰时，默认继承object类。

final关键字

final以为最终的、最后的，final可修饰类、方法、变量 。

则被final关键字修饰的类不可被继承、方法不能被重写、变量不能被重新赋值，形如常量。

多态

什么是多态

指的是事物的多种表现形态

多态的前提条件

要有继承或者实现的关系 狗继承动物类
要有方法重写 狗重写eat方法
要有父类引用指向子类对象 Animal d=new Dog();

多态的成员访问特点

成员变量：访问父类成员变量-编译、运行都看父类
成员方法：编译看父类，运行看子类。-方法重写需要父类有方法，实际执行子类的重写方法。

多态的好处和弊端

好处：提高代码的扩展性和维护性。
弊端：无法使用子类特有的成员。-必须在父类声明方法后，子类才可重写，才可访问。

多态的使用场景

可以作为方法的参数和返回值使用，可以提高代码的扩展性。

对象的类型转换

什么是对象转型

向上转型

父类引用指向子类 Animal a=new Dog();

向下转型

由父类引用转成一个对应的真实的子类对象。-解决多态中无法使用子类特有成员

注意事项

一定要确保转换的类型相同。否则会发生类型转换异常。

解决方法：关键字instanceof

用于判断左边的引用是否是右边的对象类型。

Lambda表达式

前提条件

必须是接口，接口中只能有一个抽象方法。

使用格式

(数据类型 参数名1,数据类型 参数名2,...)->{方法体}

()：代表要执行接口中对应的抽象方法。如果方法中有参数，就传递参数。

–>：固定格式。作用是将小括号中的参数传递给后面的大括号方法体中

{}：代编的是实现接口中的抽象方法后的方法体

省略规则

不建议新手使用

如果方法中只有一个参数，那么可以省略数据类型和小括号

如果方法中有多个参数，那么可以省略数据类型

如果大括号中只有一行代码，那么可省略大括号、return、分号。若省略则同时省略。

函数式接口

由于Lambda表达式只适用于仅有一个抽象方法的接口。故引入函数式接口。

方法引用与构造器引用–Lambda

作用

进一步简化Lambda表达式的书写，本质是对Lambda表达式的主体部分已存在的方法进行直接引用。

种类 Lambda表达式示例 对应的引用示例

对象名引用方法 (x,y,…)->对象名.实例方法名(x,y…) 对象名::实例方法名
构造方法引用 (x,y,…)->new 类名(x,y…) 类名::new

与throws的区别

throws关键字用在方法声明中，用来指明方法可能抛出的多个异常。

throw关键字用于方法体内，并且抛出的是一个异常类对象。

自定义异常

Java中提供的异常不符合程序的异常情况，不满足程序开发需求。Java允许用户自定义异常，但自定义的类必须继承自Exception或其子类。

垃圾回收

由于对象成为垃圾后仍会占用内存空间，故引入垃圾回收机制。在垃圾堆积到一定程度时，Java虚拟机会自动进行垃圾回收。

然而Java的自动回收垃圾具有不确定性，故，Java还提供了手动的垃圾回收方法，强制通知系统进行垃圾回收。此方法仍具有不确定性，垃圾回收可能不会立即执行。
